## v26 打包 — 修复 MODULE_NOT_FOUND（next.config.ts → next.config.js）

**问题**：v25 部署后服务器日志出现 `MODULE_NOT_FOUND`，调用栈为 `next.config.ts → transpile-config.js → config.js → next.js`。Next.js 16 在服务器启动时需要 transpile `next.config.ts`，自动尝试安装 `typescript`（devDependency），但 `pnpm install --prod` 不会安装 devDependencies。

**解决方案**：将 `next.config.ts` 改为 `next.config.js`（纯 JS），避免服务器端需要 TypeScript transpile。
- 删除 `next.config.ts`
- 创建 `next.config.js`（CommonJS 格式）
- 大小：7.77MB

**部署命令**（与 v22/v25 一致）：
```bash
rm -rf .next dist node_modules
unzip -o coze-deploy-v26.zip
pnpm install --prod --prefer-frozen-lockfile
pm2 delete cozeZnt 2>/dev/null; pm2 start dist/server.js --name cozeZnt
```

## v27 打包 — 修复 /uploads/ 静态文件 404（音频无法预览播放）

**问题**：
1. 上传的音频/视频/图片文件在服务器上存在（`public/uploads/`），但浏览器访问返回 404
2. 音频文件无法在页面内预览播放
3. 智能体提示"调用工具时出现了超时错误"

**根因**：Next.js 16 生产模式（非 standalone output）下，`public/` 目录不会被自动复制到 `.next/` 构建产物中。自定义 server（`dist/server.js`）中 Next.js 的 `getRequestHandler` 无法找到 `public/uploads/` 下的文件。

**解决方案**：在 `dist/server.js` 中添加静态文件中间件，拦截 `/uploads/` 请求，直接从 `public/uploads/` 目录读取文件并返回。
- 支持 Range 请求（HTTP 206，用于音频/视频 seek 播放）
- 正确的 MIME 类型设置
- 目录遍历攻击防护
- Cache-Control 头（24h 缓存）
- 修改文件：`src/server.ts`
- 大小：7.77MB

**部署命令**（与之前一致）：
```bash
rm -rf .next dist node_modules
unzip -o coze-deploy-v27.zip
pnpm install --prod --prefer-frozen-lockfile
pm2 delete cozeZnt 2>/dev/null; pm2 start dist/server.js --name cozeZnt
```

**重要**：部署后需要确保服务器上存在 `public/uploads/` 目录（会自动创建），且上传的文件路径正确。如果之前上传的文件丢失，需要重新上传。
