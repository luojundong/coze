## 小程序链接有效期检测（6/30 再续）
- **用户反馈**：小程序中链接复制到浏览器打开报签名错误 403，Web 端却可以正常播放
- **根因**：TOS/CDN 临时签名 URL 有 24 小时有效期（X-Tos-Expires=86400），小程序历史消息里的链接过期后 TOS 拒绝访问（`AuthorizationQueryParametersError`），Web 端可能是实时刷新 URL
- **修复（方案 4）**：`openLink` 函数重构
  1. 点击链接先发出 HEAD 请求检测有效性（8 秒超时）
  2. 200 OK → 正常复制链接 + 提示浏览器打开
  3. 403/401 → 提示"链接已过期，请重新发送指令生成"
  4. 网络错误/CORS → 降级到原逻辑直接复制（并提示"无法检测链接状态"）
- **新增方法**：`_doCopyLink(url, isValid)` 和 `_showExpiredLink(url, statusCode)`
- **修改文件**：`miniprogram/pages/tool-detail/tool-detail.js`

## 小程序链接彻底修复（7/1）
- **用户反馈**：方案 4 修复后链接还是不能复制/不能打开
- **真正根因**：
  1. TOS 域名 `coze-dianbo.tos-cn-beijing.volces.com` 未在小程序后台 request 合法域名中，HEAD 请求被微信拦截直接 fail
  2. 即使复制成功，URL 里的临时签名（X-Tos-*）24 小时后过期，浏览器访问报 `AuthorizationQueryParametersError`
  3. wxml 的 `data-url` 中包含大量 `&%?=` 等特殊字符，可能存在 dataset 解析问题
- **彻底修复**：
  1. **取消 HEAD 检测**（避免域名未配置导致直接走 fail 分支）
  2. **单击 → 直接复制完整 URL** + 弹窗提示"链接含临时签名24小时有效"
  3. **长按 → 复制"永久链接"**（去掉 `?` 后的所有签名参数），用户可拿这个永久 ID 让后端换新签名
  4. **wxml dataset 改名**：`data-url` → `data-full-url`（避免可能的 dataset 干扰）
- **修改文件**：`miniprogram/pages/tool-detail/tool-detail.wxml`、`tool-detail.js`
- **遗留问题**：永久链接不带签名，浏览器无法直接打开，需要后端配合实现"用永久 ID 换新签名 URL"接口

## 后端 refresh-url API + 小程序完整刷新链路（7/1）

- **小程序改造**：

  - `copyLinkPermanent`（长按）：同样优先调 refresh API，失败降级复制永久链接
  - 新增公共方法：`_refreshFileUrl(conversationId, filePattern, isVideo)` — 调用后端刷新
  - 新增公共方法：`_copyLinkToClipboard(url, isVideo, status)` — 统一复制/弹窗入口
- **域名配置结论**：**不需要**把 TOS 域名加到小程序合法域名，小程序只调自己的后端 `https://coze.mooibi.com`，TOS 链接通过剪贴板在外部浏览器打开
