## 激活码 Bug 修复

**问题**：用户激活工具后，打开工具页面仍提示需要输入激活码。

**根因**：`src/app/tools/[id]/page.tsx` 第 94 行和第 106 行使用了 `data.is_activated`，但 API `/api/tools/[id]` 返回的结构中 `is_activated` 嵌套在 `data.tool.is_activated` 里，顶层不存在该字段。

API 返回结构：
```json
{
  "tool": { "is_activated": true, ... },
  "coze_connected": ...,
  "is_full_access": ...
}
```

前端代码 `setTool({ ...t, is_activated: data.is_activated ?? false })` 中 `data.is_activated` 永远是 `undefined`，导致 `?? false` 后覆盖了 `t.is_activated` 的真实值。

**修复**：将 `data.is_activated` 改为 `t.is_activated`（`t` 是 `data.tool` 的解构结果）。

修改文件：`src/app/tools/[id]/page.tsx`
- 第 94 行：`data.is_activated` → `t.is_activated`
- 第 106 行：`data.is_activated` → `t.is_activated`

已构建并打包 `next-build.zip` (6.32MB, 18:20)，待用户上传到服务器部署。

**第二次排查**：用户反馈修复后仍然提示需要激活码。怀疑问题可能是：
1. 之前上传的 `next-build.zip` 未被正确部署/重启 PM2
2. JWT 中 userId 与 user_activations.user_id 不匹配（需查数据库确认）

**新增调试日志**：
- `api/tools/[id]/route.ts`：打印 userId、toolId、activations 查询结果、isToolActivated
- `api/activate/route.ts`：打印 userId、activationCode.id、codeToolId

部署后通过 `pm2 logs cozeZnt` 可查看实际执行的 userId 和激活记录。

## 小程序改造（miniprogram/）

**问题**：小程序使用 Supabase Auth 直接登录（`/auth/v1/signup`），JWT token 中 `sub` 是 `auth.users.id`；而 Web 端自定义 JWT 的 `sub` 是 `users.id`。两套 user_id 体系导致 `user_activations.user_id` 不匹配，已激活的工具在小程序中仍提示未激活。

**改造内容**：
1. `pages/login/login.js`：改用 `/api/auth/login` 和 `/api/auth/register`（自定义认证 API）
2. `app.js`：移除 Supabase 依赖（supabaseUrl/supabaseAnonKey/initConfig），简化 request 方法，错误对象附加 needActivation/needCozeAuth 标记
3. `pages/tool-detail/tool-detail.js`：修复 `is_activated` 判断（使用 `tool.is_activated`），添加 needActivation 错误处理
4. `pages/tools/tools.js`：API 返回 `is_activated`，映射为 wxml 使用的 `activated` 字段
5. `pages/index/index.js`：`status.activated` → `status.activation?.isActive`
6. `pages/login/login.js`：同上修复
7. `pages/profile/profile.js`：修复 credits 字段映射（total_granted→totalGranted, total_consumed→totalConsumed）
