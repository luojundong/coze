### V12 第三次修正 — 实际验证 Token 有效性（不再仅依赖 expiresAt）
- 问题：用户反馈仍显示"Coze 已连接"，对话框能用但不调用工具。原因是 token 的 `expiresAt` 未到期，但 token 已被 Coze 服务端撤销
- 根因：所有接口仅通过 `expiresAt` 判断 `cozeConnected`，没有实际验证 token 是否真的被 Coze API 接受
- **修复策略**：`/api/tools/[id]` 始终用用户 token 实际调一次 Coze Bot API（即使数据库有完整预设内容）
  - Bot API 返回 200 + code=0 → token 有效，`cozeConnected = true`
  - Bot API 返回 401 → token 无效，`cozeConnected = false`，**同时将数据库 `token_expires_at` 设为过去时间**
  - 数据库标记后，所有其他接口（dashboard、工具列表）都能感知到 token 已过期
  - 403/404/超时 等不改变 cozeConnected 状态
- 修改文件：`src/app/api/tools/[id]/route.ts`
  - 导入 `execute` 用于更新数据库
  - token 验证逻辑重构：先假设有效 → 用 Bot API 实际验证 → 401 时标记数据库
  - 即使数据库有完整预设内容，仍然发起验证请求（确保 token 状态准确）
- 构建：Next.js 成功
- **需重新打包**: `coze-deploy-v12.zip`（覆盖旧版本）
