# 2026-07-02

## 修复小程序任务完成后输入框/发送按钮卡住的问题

### 根本原因
`running` 状态在某些情况下无法被正常重置为 `false`：
- SSE 连接收到内容后长期保持打开，未触发 `success` 回调
- 降级轮询的 `task/status` 持续返回 pending，未进入 completed/failed 分支
- 用户离开页面后返回，状态恢复异常

`running: true` 会导致：
- 输入框 `disabled`（无法输入）
- 发送按钮显示"发送中..."且不可点击
- `sendParamForm` / `sendChat` 开头 `if (this.data.running) return` 直接拦截

### 修复内容
**miniprogram/pages/tool-detail/tool-detail.js**:
- 新增 `runningGuardTimer` 字段
- `cleanupTimers()` 统一清理 `runningGuardTimer`
- `sendChat()` 启动任务时设置 5 分钟绝对守护定时器：无论任何回调是否触发，超时后强制释放 `running`
- SSE 无数据监控定时器增加 30 秒安全网：已收到内容但 30 秒无新数据 → 结束流式并释放 `running`
- `onShow()` 增加 stuck running 检测：若 `running=true` 但无活跃定时器/连接，自动重置
