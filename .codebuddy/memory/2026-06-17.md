## V17 — 管理后台工具积分支持设置为0

### 问题
管理后台添加/修改工具时，每次消耗积分的 `min` 属性为 `1`，且 `parseInt(x) || 1` 会把 `0` 兜底为 `1`，导致无法设置免费工具（积分为0）

### 修复
`src/app/admin/tools/page.tsx`：
- 新增表单 `min="1"` → `min="0"`，编辑表单 `min={1}` → `min={0}`
- `parseInt(formCreditCost) || 1` → `isNaN(parseInt(formCreditCost)) ? 1 : parseInt(formCreditCost)`（仅 NaN 时兜底为 1，0 不再被误判）
- 编辑提交同理

## V18 — 公告/教程详情链接可点击 + 纯文本URL识别

### 问题
V16 的公告/教程详情使用 `rich-text` 渲染，但小程序 `rich-text` 中的 `<a>` 标签不响应点击；且纯文本 URL（如 `https://example.com`）未被识别为链接。

### 小程序端修复
**方案**：废弃 `rich-text` 方案，改为将内容解析为结构化 block 数组，用 `wx:for` + `<text>` 渲染，链接用独立的 `<text class="link" bindtap="openLink">` 渲染。

1. `miniprogram/pages/index/index.js`：
   - `announceDetailContent` → `announceDetailBlocks`
   - 新增 `parseContentBlocks()`：先处理 `<a>` 标签替换为占位符，再处理 `<br>`/`\n` 分段，最后识别占位符和纯文本 URL（正则：`https?://[^\s,，。,.\n\r]+`）
   - 新增 `parseLineToBlocks()`：在行内识别占位符和 URL
   - 新增 `openLink()`：点击链接复制到剪贴板 + 提示"请在浏览器打开"
2. `miniprogram/pages/index/index.wxml`：公告详情弹窗用 `wx:for` 遍历 `announceDetailBlocks`，分别渲染 text / link / br
3. `miniprogram/pages/index/index.wxss`：新增 `.announce-detail-text` / `.announce-detail-link` / `.announce-detail-br` 样式
4. `miniprogram/pages/tutorial/tutorial.js`：同样改造（复用 `parseContentBlocks` / `parseLineToBlocks` / `openLink`）
5. `miniprogram/pages/tutorial/tutorial.wxml`：教程弹窗同样改为 block 遍历
6. `miniprogram/pages/tutorial/tutorial.wxss`：新增 `.tutorial-detail-text` / `.tutorial-detail-link` / `.tutorial-detail-br`

### 管理后台
`src/app/admin/announcements/page.tsx`：公告内容 textarea 的 placeholder 改为提示支持 HTML 链接格式

### 构建说明
Web端（Next.js）修改后需要重新构建（`pnpm build`）；小程序端修改后只需在微信开发者工具中重新编译即可，无需额外构建步骤。

## V22 — 修复URL正则截断问题（飞书链接复制不全）

### 问题
URL匹配正则 `[^\s，。,.\n\r]+` 把英文句号 `.` 和中文逗号 `，` 都当作终止符，导致类似 `feishu.cn/wiki/...` 的链接在遇到 `.cn` 的 `.` 时被截断，只复制到 `https://riia7sm2p03`。

### 修改
- `miniprogram/pages/index/index.js` 第233行：正则 `[^\s，。,.\n\r]+` → `[^\s，。\n\r]+`（移除 `.` 和 `，` 的限制）
- `miniprogram/pages/tutorial/tutorial.js` 第80行：同样修复
