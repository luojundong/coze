## V9 — 修复 COLLATION 不匹配导致 SQL 查询失败

### 根因
PM2 错误日志显示：`Illegal mix of collations (utf8mb4_unicode_ci,IMPLICIT) and (utf8mb4_general_ci,IMPLICIT) for operation '='`

- `users` 表使用 `utf8mb4_unicode_ci`
- `user_memberships` 表使用 `utf8mb4_general_ci`
- LEFT JOIN 时 `u.id = um.user_id` 两个字符串列 collation 不同，MySQL 拒绝比较

### 修复
- `src/app/api/admin/memberships/route.ts`：所有 JOIN 条件添加 `COLLATE utf8mb4_general_ci`
  - `ON um.user_id COLLATE utf8mb4_general_ci = u.id COLLATE utf8mb4_general_ci`
  - 移除了 try-catch 回退机制（因为不是语法问题，是 collation 问题，回退也不会成功）
- 重新构建打包 coze-deploy-v5.zip

### 服务器部署
1. rm -rf /www/wwwroot/cozeZnt/.next /www/wwwroot/cozeZnt/dist
2. 上传 coze-deploy-v5.zip
3. unzip -o coze-deploy-v5.zip && pnpm install && pm2 stop cozeZnt && pm2 start cozeZnt
   （注意：pm2 restart 可能不重新加载模块缓存，需用 stop + start）
