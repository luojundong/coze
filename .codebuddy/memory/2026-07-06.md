## 小程序端工具详情发送/复制修复
- 问题：小程序 `tool-detail` 页面同时启动 SSE 流式 + 兜底轮询，导致一次用户发送会调用两次 workflow API；双模式竞态又容易使 `running`/`isStreaming` 状态卡住，表现为发送按钮无反应、生成的链接下方不显示"复制"按钮。
- 修复：
  1. 删除 `startDualMode`，`sendChat` 只启动 SSE 流式。
  2. SSE 失败、120 秒硬超时或完全无内容时，再由 SSE 内部启动 `startFallbackPoll`，避免同时调用两次 API。
  3. SSE 失败但已收到内容时直接结束并释放 `running`；硬超时有内容时同样释放 `running`。
  4. 兜底轮询失败/异常/初始化失败时，若 SSE 已显示内容，只把 `isStreaming` 设为 `false`，不再覆盖已有结果，确保复制按钮出现。
  5. **兜底轮询创建新任务也带上 `idempotency_key`**，与 SSE 共用同一个 key，防止 SSE 失败后兜底轮询重复扣费。
  6. **收到 `conversation.message.completed` / `done` / `[DONE]` 事件时立即结束流式**：设置 `isStreaming: false` 和 `running: false`，并中断 SSE 连接，保证生成完整链接后立刻显示"复制"按钮且能重新发送。
  7. 快捷参数表单的"点击发送"按钮也跟随 `running` 状态显示"发送中..."和禁用样式，与输入区发送按钮保持一致。
- 修改文件：`miniprogram/pages/tool-detail/tool-detail.js`、`miniprogram/pages/tool-detail/tool-detail.wxml`、`miniprogram/pages/tool-detail/tool-detail.wxss`
