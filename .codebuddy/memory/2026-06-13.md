## 手机号注册失败问题排查
- 用户反馈用手机号注册提示"注册失败，请稍后重试"
- 排查代码逻辑无误（auth.ts createUser 正确识别手机号，前端正确发送请求）
- 根本原因：服务器数据库 `users` 表可能尚未执行 v3 迁移（缺少 `phone` 列），导致 INSERT 报错进入 catch 块
- 需要在服务器执行：`ALTER TABLE users ADD COLUMN phone VARCHAR(20) NULL AFTER email;` 和唯一索引
- 优化了 register/route.ts 的 catch 日志，增加 error.message 和 error.stack 输出便于后续排查
- 重新构建并打包 next-build.zip

## 激活码详情页适用工具显示修复
- 问题：激活码详情页 (`codes/[batchId]`) 的"适用工具"永远显示"全部工具"
- 原因：详情页接口定义的是旧字段 `tool_id`(单数)/`tool_info`(单数)，但 API 实际返回的是 `tool_ids`(复数)/`tool_infos`(复数数组)
- 修复：将 `CodeItem` 和 `BatchInfo` 接口字段改为 `tool_ids` 和 `tool_infos`；渲染改用 `getToolNamesDisplay()` 函数显示工具名称列表
- 构建成功，输出 next-build.zip

## OAuth 配置变更后旧 Token 未清除问题
- 问题：修改 Coze Client ID/Secret 后，旧工具仍可用、新工具不可用
- 原因：Token 是用户绑定的（coze_tokens 表），修改全局 OAuth 配置不会自动清除旧 token
- 修复：`api/admin/oauth-config PUT` 中，当 client_id 或 client_secret 变更时，执行 `DELETE FROM coze_tokens` 清除所有用户 token
- 前端 oauth 页面：保存成功提示增加"所有用户的 Coze 连接已重置，用户需要重新授权"提醒
- 构建成功，输出 next-build.zip
