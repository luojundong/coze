# 2026-07-11 工作日志

## 修复：小程序分享码扫码进入但不绑定分销（09:21）
用户反馈：体验版本就能生成分享二维码、体验用户也能扫码进小程序，只是"不在分销里"。
根因：`miniprogram/app.js` 完全没有处理扫码带来的 `scene`/`ref`。`login.js` 只在页面自身 `onLoad(options)` 里读 `options.scene` 设 `pendingRef`。但当小程序**已在后台运行**时被扫码唤醒，微信只触发 `App.onShow(options)`，`login` 页面 `onLoad` 不再执行 → `pendingRef` 始终为空 → 注册时不发绑定请求 → 进得来但不在分销里（最常见的真实分享场景）。

修复：在 `app.js` 增加 `captureReferral(options)` 方法，在 `onLaunch` 和 `onShow` 中都调用，从 `options.ref` / `options.query.ref` / `options.scene` 提取分销码写入 `globalData.pendingRef`（scene 做 decodeURIComponent 兜底）。`login.js` 的 `handleSubmit` 本来就读 `app.globalData.pendingRef` 并在绑定后置空，无需改动。

注意点与结论：
- 后端 `/api/referral/link` (POST) 绑定逻辑本身正确，无需改后端。
- 改动仅小程序端 `app.js`，必须**重新上传小程序**（先传体验版验证，再传正式版走方案1）。
- 测试必须用**第二个/新账号**扫码注册（不能自己扫自己的码，会"不能推荐自己"被拒，属正常）。
- 方案1（发布正式版 + v43 release 小程序码）仍推荐用于真正面向公众的推广；本修复对 trial 与 release 都必需。
