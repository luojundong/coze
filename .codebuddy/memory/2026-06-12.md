## 第18轮修复（深度根因分析：多轮对话超时彻底解决）

### 根因分析（三层架构缺陷）

1. **Next.js 默认超时限制**（最关键）：`task/run` 和 `task/status` 路由未设置 `maxDuration`，Next.js 自托管默认约 60 秒执行上限。`executeCozeTask` 内部轮询 Coze（40次×1.5s=60s）超过限制时，Next.js 终止请求的异步上下文，task 永远到不了 `completed`。

2. **Fire-and-forget Promise 可能被 GC 回收**：`executeCozeTask` 不 await，但未被任何引用持有。Node.js 在请求结束后可能回收未被引用的 Promise 链。

3. **小程序轮询逻辑缺陷**：轮询 `catch` 中无法区分"后端返回 failed"和"网络抖动"，failed 错误被当作网络错误吞掉继续重试。

### 修复内容

**后端 `task/run/route.ts`**：
- 新增 `export const maxDuration = 120`（120 秒执行上限）

## 第20轮修复：SSE 结束后内容消失 bug

### 问题现象
AI 内容正常流式渲染完后，弹窗"未收到回复"，已显示的文字清空消失。

### 根因（豆包分析确认）
1. **定时器残留**：SSE 流正常结束后，`noDataTimer` 仍在后台运行，触发降级逻辑 → `startFallbackPoll` 覆盖已渲染内容
2. **无任务互斥**：没有 `currentTaskId`，旧请求/旧定时器继续执行，篡改新页面数据
3. **`success` 回调误判**：`accumulatedContent` 为空时设置为"未收到回复"（闭包可能未拿到 onChunkReceived 中累积的值）
4. **降级不检查已有内容**：`fail` 回调和 `noDataTimer` 触发降级时，不判断 `hasReceivedChunk`

### 修复方案（完整重写 `tool-detail.js` v2）

**核心机制**：
- `currentTaskId` 互斥：每轮对话生成唯一 `taskId`，所有回调/定时器只处理当前 taskId
- `hasReceivedChunk` 标记：只要收到过任意一段文字，后续任何失败/降级逻辑都不清空已有内容
- `streamFinished` 标记：SSE 已正常结束/已降级后，忽略所有后续 `onChunkReceived` 事件
- `noDataTimer` 存入 data：纳入 `cleanupTimers()` 统一管理，sendChat 入口强制清理

**关键修复点**：
1. `sendChat()` 入口：每次发送前 `cleanupTimers()` 销毁上一轮所有定时器/连接
2. `noDataTimer` 10s 无数据 → 先判断 `hasReceivedChunk`，已有内容则不降级直接结束
3. `success` 回调：先判断 `hasReceivedChunk`，有内容则保留 accumulatedContent
4. `fail` 回调：先判断 `hasReceivedChunk`，有内容则不清空直接结束
5. `startFallbackPoll` 所有分支：`taskId !== currentTaskId` 立即停止
6. 降级轮询中：`status=pending` 只轮询不修改文字，`hasPollContent=true` 时失败不清空

**降级触发条件调整**：
- SSE 无数据超时：5s → **10s**（减少误降级）
- 已收到过内容时 SSE 无数据 → 直接结束（不降级），保留已有内容
