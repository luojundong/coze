## Web 端上传图片后 Coze 报“下载图片链接超时”问题（v46 修复）
- 根因分析：服务器托管 `/uploads/` 本身正常（浏览器能加载，server.ts 静态处理支持 Range/Content-Length 正确）。超时来自 **Coze 工作流的图片下载节点**对外部 URL 有体积/超时限制；用户手机拍的“风景照片”常为 5-10MB，传给 Coze 时超过其下载超时即报错。Coze 自身输出在 TOS(`*.volces.com`) 上下载无此问题，说明是“大图 + 我们的域名”组合导致。
- 项目无自有 TOS 桶（`@aws-sdk/client-s3` 是依赖但 src 内无任何 S3 上传调用），也无图片压缩库（sharp/jimp 均不在依赖，且本环境原生编译脆弱不宜新增），故采用**浏览器端上传前压缩**方案。
- 新增 `src/lib/image-compress.ts`：`compressImageFile`（canvas，仅 image/*，<1.5MB 跳过，>2000px 等比缩放，输出 JPEG 0.85，失败回退原图）。
- `src/app/tools/[id]/page.tsx` 的 `handleUpload`：当 `mediaType==='image'` 时先压缩再 `formData.append('file', ...)` 上传到 `/api/upload/media`。典型 10MB→<500KB，Coze 秒级下载。
- 注：此修复覆盖 Web 端工具图片上传（报告场景）。小程序端若也有同类超时，可用 `wx.compressImage` 类似处理（本次未做，因报告为 Web 端）。若压缩仍偶发超时，根因可能是 Coze 到我们服务器的网络可达性，需将输入图托管到 CDN/TOS（需额外凭据）。

## 修复：管理后台审计日志「内容加载失败」（v47）
