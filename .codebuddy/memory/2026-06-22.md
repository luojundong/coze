# 2026-06-22

## 修复工具调用超时问题 — V16

### 问题
用户调用工具（快捷输入上传声音/视频/图片）时频繁出现"发送失败: 智能体响应超时，请稍后重试"

### 根本原因分析
1. **`callAuthenticatedApi` 无超时控制**：`src/lib/api-client.ts` 直接使用 `fetch()` 无 `AbortController`，所有请求可能无限挂起
2. **服务端 `workflow/stream` SSE 读取无超时**：`cozeReader.read()` 无超时保护，Coze 卡住时前端会一直等
3. **上传 API 缺少 `maxDuration`**：大文件上传可能触发 Next.js 默认超时
4. **同步轮询模式 `workflow/run` 效率低**：Bot 类型 30s 轮询太短，图片/音视频分析不够
5. **Web 端降级轮询不复用 SSE 的 taskId**：每次都创建新任务浪费积分

### 修复内容

#### Web 端服务端
- `src/lib/api-client.ts`：添加 `AbortController` 超时控制，默认 30s，支持 `timeout` 参数（0=不设超时）
- `src/app/api/upload/media/route.ts`：添加 `maxDuration = 180`（大文件上传）
- `src/app/api/workflow/stream/route.ts`：Coze SSE 读取添加 20s 超时 + 6次连续超时自动终止
- `src/app/api/workflow/run/route.ts`：轮询从 20次×1.5s=30s → 40次×2s=80s
- `src/lib/coze-token.ts`：Token 刷新添加 10s AbortController 超时

#### Web 端前端
- `src/app/tools/[id]/page.tsx`：
  - SSE 调用设置 `timeout: 0`（不设超时，依赖 SSE 自身检测）
  - 降级轮询复用 SSE 注入的 taskId（避免重复创建任务）
  - 文件上传添加 120s 超时
  - 添加 task_id 事件捕获（与小程序端一致）

#### 小程序端
- `miniprogram/pages/tool-detail/tool-detail.js`：
  - SSE 无数据超时：10s → 20s（与 Web 端统一）
  - SSE 硬超时：90s → 120s
  - 文件上传 `wx.uploadFile` 添加 `timeout: 120000`
  - 降级轮询硬超时：60s → 90s
