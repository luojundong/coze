## 小程序工具使用超时问题修复（2026-07-07 10:56）

### 问题现象
- 小程序使用工具一直失败，弹窗显示"任务响应超时"
- 错误提示从未在对话框里显示
- 7月6号之前一直正常，添加"幂等键防重复扣费"之后开始出问题

### 根因分析（两个关键 Bug）

#### Bug 1：微信小程序 Response Header 大小写问题（`tool-detail.js:977`）
- 代码 `res.header['Content-Type']` 在微信小程序中永远为 `undefined`
- 微信小程序 `wx.request` 返回的 Header key **全部是小写**，实际是 `res.header['content-type']`
- 导致 JSON 错误响应无法被识别 → 错误信息被丢弃 → 进入兜底轮询 → 一直失败 → 5分钟后才弹 timeout toast

#### Bug 2：数据库 `idempotency_key` 列不存在（`credit.ts`）
- 迁移文件 `20260706_add_idempotency_key.sql` 未被自动执行
- `deductCredits()` 中 `SELECT ... WHERE idempotency_key = ?` 直接抛出 MySQL 异常 `Unknown column 'idempotency_key'`
- 导致所有工具调用在扣费环节失败

### 修复内容

#### 1. `src/lib/credit.ts` — 优雅降级
- `deductCredits()` 中的幂等键 SELECT 查询包裹 try-catch，捕获 `Unknown column` 错误时跳过幂等检查，继续正常扣费
- INSERT 事务记录时同理：列不存在则重试不带 `idempotency_key` 列的 INSERT
- 打印 warning 日志提示运行迁移

#### 2. `miniprogram/pages/tool-detail/tool-detail.js` — 三处修复
- **Header 大小写**：`res.header['content-type']` 优先于 `res.header['Content-Type']`
- **状态码兜底**：新增 `isErrorResponse` 判断（`statusCode >= 400 && res.data` 为对象时直接当 JSON 错误处理）
- **onChunkReceived 中检测 JSON 错误**：在 `enableChunked` 模式下，JSON 错误可能通过 chunk 到达而非 `res.data`。新增检测逻辑：若第一个 chunk 不包含 `event:`/`data:` 且解析为 JSON 且含 `error` 字段 → 立即显示错误并释放 running
- **兜底错误匹配**：当没有任何已知错误类型匹配时，若 `statusCode >= 400`，直接显示状态码+data 作为错误消息

#### 3. `src/server.ts` — 自动迁移
- 服务启动时自动检查 `credit_transactions` 表：
  - 是否有 `idempotency_key` 列（通过 `INFORMATION_SCHEMA`）
  - 是否有索引 `idx_credit_idempotency_key`
  - 是否有唯一约束 `uniq_credit_idempotency_key`
- 任一缺失则自动执行 `ALTER TABLE` 添加
- 所有操作均在 `server_startup_at` 写入之后的启动阶段完成

## v39: 工作流生成媒体自动下载到 /public/download/（2026-07-07 14:44）

### 需求
用户生成出来的视频或图片，在后台自动下载保存到云文件夹 public/download/，管理员每天手动清理。

### 实现方案

#### 新增文件
- **src/lib/media-downloader.ts**: 通用媒体下载工具
  - extractMediaUrls(content): 从 Coze 响应 content（字符串/对象/JSON）中提取媒体 URL
  - collectMediaFromSSEEvent(data, collectedUrls): 从 SSE 事件中收集媒体 URL
  - collectMediaFromMessages(messages): 从消息列表中收集媒体 URL
  - triggerBackgroundDownload(taskId, urls, label): fire-and-forget 批量下载
  - 特性: 并发控制(MAX_CONCURRENT=3)、单文件 120s 超时、自动去重、自动创建目录
  - 支持格式: mp4/mov/avi/mkv/webm/wmv/flv/m4v/png/jpg/jpeg/gif/webp/bmp/svg

#### 修改文件
1. **src/app/api/workflow/stream/route.ts**
   - Bot SSE 流式: 在事件解析循环中 collectMediaFromSSEEvent()，流结束后 triggerBackgroundDownload()
   - Bot 非流式 JSON: 在消息提取后 collectMediaFromMessages() + triggerBackgroundDownload()

2. **src/app/api/workflow/task/run/route.ts**
   - executeCozeTask() SSE 解析: 同样收集媒体 URL，流结束后触发下载

3. **src/app/api/workflow/run/route.ts**
   - Bot 同步轮询: 消息解析后收集媒体 URL 并触发下载

4. **.gitignore**: 新增 public/download/ 忽略规则

### 设计关键
- fire-and-forget 模式: 不阻塞 SSE 响应，不 await 下载结果
- 用户无感知: 文本回复照常返回，媒体后台静默落盘
- 每天手动清理: rm -rf public/download/*

- coze-deploy-v39.zip（7.39 MB, 1278 files）
- TypeScript 编译通过，83 routes，零错误
