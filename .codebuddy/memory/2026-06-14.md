## 根因确认：GT2图片生成 bot 未发布到 Agent As API 渠道
- 添加日志后发现：Coze 返回 HTTP 200 但 `content-type: application/json`（非 SSE），旧代码只处理 SSE 流导致"未收到回复"
- Coze 返回的具体错误：`{"code":4015,"msg":"The bot_id 7651068062136926260 has not been published to the channel Agent As API."}`
- 需要在 Coze 后台将该 bot 发布到「Agent as API」渠道
- 修复 `src/app/api/workflow/stream/route.ts`：
  - 增加非 SSE（JSON）响应的处理分支：检测 content-type 包含 `application/json` 且不包含 `text/event-stream` 时，解析 JSON 提取内容
  - 从 `data.messages` 中提取 assistant 的 answer 消息
  - 对错误码（code != 0）返回 msg 中的错误信息
  - 以 SSE 格式返回给前端保持兼容
  - 同时更新 taskStore 供后台轮询
- 最终修复：在非 SSE JSON 处理分支中，优先判断 code !== 0 作为错误响应，显示明确错误信息
- 根因最终确认：GT2图片生成 在 Coze 后台未勾选「Agent as API」发布渠道，导致 code: 4015
- 用户确认已补勾选 API 渠道发布

## 图片参数支持（URL + 本地上传）
- 用户要求：快捷输入支持图片，可以是链接也可以是本地上传
- 关于 Coze 能否找到上传图片的分析：
  - Coze 智能体通过文本消息接收参数，无法直接访问用户本地的文件
  - 图片必须转换为公网可访问的 URL，Coze 智能体才能识别和处理
  - 方案：本地上传 → Supabase Storage（public bucket）→ 拿到公网 URL → 拼接在文本参数中传给 Coze
- 修改文件：
  1. `src/app/admin/tools/page.tsx`：`ParamPreset.type` 增加 `'image'` 类型，管理后台选择器增加"图片(URL/上传)"选项
  2. `src/app/api/upload/image/route.ts`（新建）：图片上传 API
     - 认证校验（verifyAuth）
     - 文件类型限制：png/jpeg/gif/webp/bmp
     - 文件大小限制：10MB
     - 上传到 Supabase Storage 的 `tool-uploads` bucket（public，首次自动创建）
     - 返回公网 URL
  3. `src/app/tools/[id]/page.tsx`：新增 `ImageParamInput` 组件
     - 两种模式切换：链接模式（输入URL） / 上传模式（本地上传）
     - 链接模式：输入框 + URL 图片预览缩略图
     - 上传模式：点击/拖放上传区域 → 上传到 `/api/upload/image` → 预览已上传图片 → 支持更换/移除
     - 上传后拿到公网 URL，最终以 `标签：URL` 格式拼接发送给 Coze
- 构建成功，新增路由 `/api/upload/image`

## Coze Token 认证失败修复
- 问题：快捷输入发送后提示"智能体调用失败: The token you entered is incorrect"
- 根因：Coze access_token 过期，refresh_token 刷新可能也失败（OAuth clientSecret 不匹配或 token 已彻底失效）
- 修复 3 处：
  1. `src/app/api/workflow/stream/route.ts`（SSE 透传路径）：`!cozeResponse.ok` 时检测 401/403/token/authentication 关键字，标记 `needCozeAuth: true` 返回给前端
  2. `src/app/api/workflow/stream/route.ts`（taskPromise 后台路径）：同样增加 token 失效检测，返回友好提示
  3. `src/lib/coze-token.ts`：`refreshCozeToken` 失败时自动 `DELETE FROM coze_tokens` 清理无效 token，避免反复尝试刷新；返回明确错误信息
- 用户需要重新连接 Coze 账户（OAuth 重新授权获取新 token）
- 构建成功，输出 next-build.zip
