### V15 — 修复公告链接不能直接打开的问题
- **问题**：Web 端公告中的 `<a href="...">` 链接无法直接点击打开
- **根本原因**：
  1. 列表预览中 `{a.content}` 渲染为纯文本，HTML 标签被原样显示
  2. 详情弹窗中 `dangerouslySetInnerHTML` 缺少 `&amp;` 和 `&#39;` 的 HTML 实体解码
- **修复方案**：
  - 添加 `stripHtml()` 工具函数：剥离 HTML 标签显示纯文本（列表预览用）
  - 详情弹窗：增加 `&amp;` → `&` 和 `&#39;` → `'` 的替换，确保链接 URL 中的特殊字符正确解码
  - 保留 `target="_blank" rel="noopener noreferrer"` 在新窗口打开
- **修改文件**：
  - `src/app/dashboard/page.tsx` — 列表预览 + 详情弹窗
  - `src/app/tools/page.tsx` — 列表预览 + 详情弹窗
